# Iteration 002 — HTTPS/TLS with certificate pinning

**Date:** 2026-03-08
**Branch:** `feat/https-tls`

## What was done

### Problem being solved

The Ollama endpoint was served over plain HTTP. Any network observer between the deployer and the EC2 instance could read prompts and responses. API key bearer tokens were also transmitted in clear text.

### Solution

Generate a self-signed TLS certificate before provisioning. Upload the cert and key to EC2 via CloudFormation user_data so Ollama listens on HTTPS. The client pins the certificate's SHA-256 fingerprint instead of trusting a CA — TOFU-style MITM protection without requiring a public CA or Let's Encrypt.

### Architecture changes

**New package: `internal/certutil/`**

| Function | Description |
|---|---|
| `GenerateSelfSigned()` | ECDSA P-256, 10-year validity, returns `certPEM`, `keyPEM`, `fingerprint` (SHA-256 hex) |
| `NewPinnedTransport(fingerprint)` | `*http.Transport` that bypasses CA chain but enforces exact fingerprint on `PeerCertificates[0]` |

The pinning checks only the leaf certificate (`PeerCertificates[0]`). Checking the full chain allowed a cert higher up to satisfy the check — the fix is to be explicit about the leaf.

**Deploy flow changes (`internal/cli/deploy.go`)**

1. `certutil.GenerateSelfSigned()` called before `prov.Deploy()`
2. `TLSCert`, `TLSKey`, `TLSFingerprint` passed into `provider.DeployInput`
3. Endpoint scheme switched from `http://` to `https://`
4. `TLSCert` and `TLSFingerprint` stored on the `Deployment` struct
5. `waitForOllama` uses `certutil.NewPinnedTransport` — no plain HTTP health check
6. Ready output prints `haven cert <id>` hint

**Provider/CFN propagation**

- `cfn.DeployInput` gained `TLSCert` and `TLSKey` fields
- `GenerateTemplate` receives them and writes the PEM files to EC2 via user_data
- `bootstrap.Generate` guards against empty TLS values
- `AWSProvider.Deploy` wires the fields from `provider.DeployInput`

**New CLI command: `haven cert <deployment-id>`**

Prints the TLS certificate PEM for a deployment to stdout. Intended usage:

```
haven cert <id> > cert.pem
SSL_CERT_FILE=cert.pem python your_script.py   # OpenAI SDK
curl -k --cacert data/certs/<id>.pem ...          # curl
```

### Security properties

- Encryption in transit: TLS 1.2+ enforced
- MITM protection: fingerprint pinned at deploy time, stored in S3 state
- No CA required: self-signed cert is fine because the fingerprint is the trust anchor
- Key never leaves AWS: private key only in EC2 memory and CloudFormation user_data (which is visible in AWS console — acceptable for this use case)
