# Iteration 009 — Interactive Chat Command

**Date:** 2026-03-09
**Branch:** `feat/chat-command`

## What was done

### Problem being solved

After deploying a model with `haven deploy`, users had to use `curl` or external tools to interact with it. There was no built-in way to chat with the model from the terminal.

### Solution

Added `haven chat [deployment-id]` — an interactive terminal REPL that connects to a deployed model via Ollama's `/api/chat` endpoint, streams responses token-by-token, and maintains conversation history for the session.

## What works

- `go build ./cmd/haven/` — compiles
- `go test -race ./...` — 9/9 packages pass (6 new tests in chat_test.go)
- `go vet ./...` — clean
- `golangci-lint run` — clean

Features:
- Auto-selects deployment if only one exists
- Prompts user to select if multiple deployments
- Streams tokens in real-time as they arrive
- Maintains in-memory conversation history (multi-turn)
- Graceful exit on `exit`, `quit`, Ctrl+D, Ctrl+C
- Error resilience: prints error, removes failed message, stays in REPL
- TLS certificate pinning via existing `certutil.NewPinnedTransport`

## What's not covered

- System prompt / `--system` flag
- Chat history persistence across sessions
- Multi-line input
- Token count / timing display
- Integration tests (requires live Ollama endpoint)
